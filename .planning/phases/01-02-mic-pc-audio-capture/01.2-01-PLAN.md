  <action>
Create Mic/main.py with the following components:

1. **AudioCapture class** with:
   - `__init__(sample_rate=16000, chunk_size=1024, device_index=None)` - Initialize audio capture
   - `start()` - Begin continuous audio capture in background thread
   - `stop()` - Stop audio capture
   - `get_audio_chunk()` - Returns latest audio chunk as bytes or None
   - `is_recording` - Property returning capture status

2. **VoiceActivityDetector class** with:
   - `__init__(sample_rate=16000, mode=3)` - Initialize VAD (mode 0-3, 3 = most aggressive)
   - `is_speech(audio_chunk)` - Returns True if voice detected, False otherwise
   - `set_mode(mode)` - Change VAD sensitivity (0=least aggressive, 3=most)

3. **AudioBuffer class** with:
   - `__init__(max_size_seconds=10)` - Initialize buffer with max duration
   - `add_chunk(audio_chunk)` - Add audio chunk to buffer
   - `get_buffer()` - Return all buffered audio as bytes
   - `clear()` - Clear buffer
   - `get_duration()` - Return current buffer duration in seconds

4. **MicController class** - Main orchestrator that combines all three:
   - `__init__(sample_rate=16000, chunk_size=1024, vad_mode=3)` - Initialize all components
   - `start()` - Start capture and VAD
   - `stop()` - Stop capture
   - `listen()` - Blocking call that waits for voice activity, returns buffered audio when speech ends
   - `get_live_audio()` - Non-blocking get latest audio chunk

5. **Main execution** when run directly:
   - Detect available audio devices
   - Initialize MicController with defaults
   - Print audio levels in real-time (for testing)
   - Handle Ctrl+C gracefully

Use sounddevice library (pip install sounddevice) - it works cross-platform and has good VAD support via webrtcvad complement.

IMPORTANT: Write production-quality code with proper error handling, logging, and type hints.
